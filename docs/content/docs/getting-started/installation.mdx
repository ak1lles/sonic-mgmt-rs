## Prerequisites

- **Rust 1.75+** and Cargo ([install via rustup](https://rustup.rs/))
- **Git** for cloning the repository
- Optional: **Docker** and **Docker Compose** for integration testing with SONiC VS
- Optional: **cargo-make** for task automation (`cargo install cargo-make`)

## Build from Source

```sh
# Clone the repository
git clone https://github.com/your-org/sonic-mgmt-rs.git
cd sonic-mgmt-rs

# Build in release mode (optimized, stripped binary)
cargo build --release
```

The compiled binary is at `target/release/sonic-mgmt`.

### Build Profile

The release build uses aggressive optimizations:

| Setting | Value | Effect |
|---|---|---|
| LTO | Enabled | Link-time optimization across all crates |
| Codegen units | 1 | Maximum optimization (slower compile) |
| Strip | Enabled | Removes debug symbols for smaller binary |

## Install the Binary

Copy the binary to a location in your `$PATH`:

```sh
# System-wide
sudo cp target/release/sonic-mgmt /usr/local/bin/

# Or user-local
cp target/release/sonic-mgmt ~/.local/bin/
```

Verify the installation:

```sh
sonic-mgmt --help
```

## Install cargo-make (Recommended)

`cargo-make` provides pre-configured tasks for building, testing, and Docker management:

```sh
cargo install cargo-make
```

Then use the included tasks:

```sh
cargo make build        # Build the project
cargo make test         # Run unit tests
cargo make docker-test  # Run integration tests against SONiC VS
cargo make ci           # Full CI: lint + tests + integration + teardown
```

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `SONIC_CONFIG` | `sonic-mgmt.toml` | Path to the main configuration file |
| `SONIC_LOG` | `info` | Log level: `trace`, `debug`, `info`, `warn`, `error` |

## First-Time Setup

After installation, run the interactive setup wizard:

```sh
sonic-mgmt init
```

This walks you through creating your initial `sonic-mgmt.toml` configuration file with connection defaults, testbed settings, and logging preferences.

## Verify with Docker (Optional)

If you have Docker installed, you can verify everything works end-to-end against a SONiC Virtual Switch:

```sh
# Start the SONiC VS container and run integration tests
cargo make docker-test

# SSH into the running SONiC VS to inspect it
cargo make docker-ssh

# Tear down when done
cargo make docker-down
```

See the [Docker Development](/docs/development/docker) guide for more details.

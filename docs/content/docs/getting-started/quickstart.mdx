title: Quick Start

---

This guide walks you through the minimum steps to go from a fresh install to running your first test against a SONiC device.

## Step 1: Initialize Configuration

```sh
sonic-mgmt init
```

This interactive wizard creates `sonic-mgmt.toml` in your working directory. It will ask you about:

- SSH connection defaults (port, timeout, retries)
- Test runner settings (parallelism, timeouts)
- Logging preferences

You can always edit this file later with `sonic-mgmt config edit`.

## Step 2: Create a Testbed

A **testbed** defines the physical or virtual devices you're testing against. Use the wizard:

```sh
sonic-mgmt testbed wizard
```

This guides you through specifying:

- **Testbed name** and group (e.g., `vms-t0` in group `vms`)
- **Topology type** (e.g., `t0`, `t1`, `dualtor`)
- **DUT devices** -- hostname, management IP, hardware SKU, platform, credentials
- **PTF container** -- image and IP address
- **Server** -- the physical server hosting the testbed
- **Neighbor VMs** -- (optional) auto-generated from topology if omitted

The wizard outputs a TOML testbed definition file.

## Step 3: Create a Device Inventory

The **inventory** is a flat catalog of all devices the framework can contact:

```sh
sonic-mgmt device wizard
```

Or manually create an inventory file (see [Configuration > Inventory](/docs/configuration/inventory)).

## Step 4: Deploy the Topology

```sh
sonic-mgmt testbed deploy my-testbed
```

This provisions the topology defined in your testbed, setting up VMs, VLANs, and network wiring.

## Step 5: Verify Health

```sh
sonic-mgmt testbed health my-testbed
```

Runs concurrent health checks on all devices in the testbed and reports their status.

## Step 6: Run Tests

```sh
# Discover available tests
sonic-mgmt test discover --path tests/

# Run all tests
sonic-mgmt test run --path tests/

# Run specific tests by tag
sonic-mgmt test run --path tests/ --tag bgp

# Run with parallelism
sonic-mgmt test run --path tests/ -j 4
```

## Step 7: View Results

```sh
# View the test results summary
sonic-mgmt test results --output output/

# Parse JUnit XML reports
sonic-mgmt report parse output/results.xml

# Upload to Kusto (if configured)
sonic-mgmt report upload output/results.xml
```

## Example: Minimal Manual Configuration

If you prefer to skip the wizards, here are the minimum files you need:

### sonic-mgmt.toml

```toml
[testbed]
active = "my-testbed"
file   = "testbed.toml"

[connection]
timeout_secs = 30

[testing]
output_dir = "output"

[logging]
level = "info"
```

### testbed.toml

```toml
name  = "my-testbed"
group = "lab"
topo  = "t0"

ptf_image = "docker-ptf"
ptf_ip    = "10.255.0.188"
server    = "server-1"
vm_base   = "VM0100"

[[duts]]
hostname = "sonic-switch-1"
mgmt_ip  = "10.0.0.1"
hwsku    = "ACS-MSN2700"
platform = "mellanox"

  [duts.credentials]
  username = "admin"
  password = "password"
```

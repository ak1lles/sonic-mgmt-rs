sonic-mgmt-rs uses TOML configuration files for all settings. There are three main configuration files:

| File | Purpose |
|---|---|
| `sonic-mgmt.toml` | Application settings (connections, testing, logging) |
| Testbed file | Testbed definition (DUTs, topology, PTF, neighbors) |
| Inventory file | Device catalog (all devices the framework can contact) |

## Configuration Layering

Settings are resolved in priority order:

1. **CLI flags** -- highest priority (e.g., `--log-level debug`)
2. **Environment variables** -- `SONIC_CONFIG`, `SONIC_LOG`
3. **Project file** -- `sonic-mgmt.toml` in the working directory
4. **Compiled defaults** -- built into the binary

## Application Config (sonic-mgmt.toml)

This is the main configuration file. Generate one interactively with `sonic-mgmt init` or `sonic-mgmt config init`.

```toml
[testbed]
active = "my-testbed"          # Which testbed to use by default
file   = "testbed.toml"       # Path to the testbed definition

[connection]
default_ssh_port = 22          # Default SSH port
timeout_secs     = 30          # Connection timeout in seconds
retries          = 3           # Number of connection retry attempts
key_path         = ""          # Default SSH private key path

[testing]
parallel_workers = 1           # Number of parallel test workers
timeout_secs     = 900         # Per-test timeout (15 minutes)
output_dir       = "output"    # Directory for test artifacts
report_format    = "junit_xml" # Output format: junit_xml, json, toml

[reporting]
backend_url  = ""              # Kusto cluster URL
auth_method  = "azure_default" # Authentication method
database     = ""              # Kusto database name
table        = ""              # Kusto table name

[topology]
vm_base_ip = "10.250.0.2"     # Starting IP for VMs
vlan_base  = 1000              # Starting VLAN ID
ip_offset  = 1                 # IP offset for allocation

[logging]
level  = "info"                # Log level: trace, debug, info, warn, error
format = "full"                # Format: full, compact, pretty, json
# file = "/var/log/sonic-mgmt.log"  # Optional log file
```

### Sections

#### `[testbed]`

- **active** -- name of the default testbed. Commands like `testbed deploy` use this unless overridden.
- **file** -- path to the testbed definition file (TOML or YAML).

#### `[connection]`

- **default_ssh_port** -- used when a device doesn't specify its own port.
- **timeout_secs** -- how long to wait for a connection before timing out.
- **retries** -- number of times to retry a failed connection.
- **key_path** -- default SSH private key. Individual devices can override this.

#### `[testing]`

- **parallel_workers** -- number of concurrent test processes. Default 1 (sequential) is safest.
- **timeout_secs** -- per-test timeout. Default 900 seconds (15 minutes).
- **output_dir** -- where test artifacts, logs, and reports are written.
- **report_format** -- output format for test results.

#### `[reporting]`

- **backend_url** -- Azure Data Explorer (Kusto) cluster URL for uploading results.
- **auth_method** -- Kusto authentication: `azure_default`, `azure_cli`, `device_code`, `managed_identity`, `app_key`, `user_token`, `app_token`.
- **database** / **table** -- Kusto destination for results.

#### `[topology]`

- **vm_base_ip** -- starting management IP for auto-generated VMs.
- **vlan_base** -- starting VLAN ID for topology generation.
- **ip_offset** -- offset added to allocated IPs.

#### `[logging]`

- **level** -- minimum log level. Can also be set via `SONIC_LOG` env var or `--log-level` flag.
- **format** -- log output format.
- **file** -- optional path to write logs to a file.
